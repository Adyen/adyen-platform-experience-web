# i18n Patterns

All user-facing text must use the internationalization system. Never hardcode strings.

## Getting i18n

```typescript
import useCoreContext from '../../../core/Context/useCoreContext'; // Path may need to be adjusted

const MyComponent = () => {
    const { i18n } = useCoreContext();

    return <span>{i18n.get('my.translation.key')}</span>;
};
```

## Translation Methods

### `i18n.get(key, options?)` - Basic Text

```typescript
// Simple key
i18n.get('transactions.overview.title');
// → "Transactions"

// With interpolation
i18n.get('transactions.count', { values: { count: 5 } });
// → "5 transactions"

// With plural forms
i18n.get('items.selected', { count: 1 });
// → "1 item selected"
i18n.get('items.selected', { count: 5 });
// → "5 items selected"
```

### `i18n.amount(value, currencyCode, options?)` - Currency

```typescript
// Basic amount (value in minor units)
i18n.amount(1234, 'EUR');
// → "€12.34"

i18n.amount(100000, 'USD');
// → "$1,000.00"

// Currencies without decimals
i18n.amount(1234, 'JPY');
// → "¥1,234"

// Negative amounts
i18n.amount(-5000, 'EUR');
// → "- €50.00"

// Hide currency symbol
i18n.amount(1234, 'EUR', { hideCurrency: true });
// → "12.34"
```

### `i18n.date(date, options?)` - Date Formatting

```typescript
// Basic date
i18n.date('2024-01-15T10:30:00Z');
// → "Jan 15, 2024"

// With custom options
i18n.date('2024-01-15T10:30:00Z', { month: 'long', year: 'numeric' });
// → "January 2024"
```

### `i18n.fullDate(date)` - Date with Time

```typescript
i18n.fullDate('2024-01-15T10:30:00Z');
// → "Jan 15, 2024, 10:30:00"
```

### `i18n.has(key)` - Check Key Exists

```typescript
if (i18n.has('optional.feature.title')) {
    return <h2>{i18n.get('optional.feature.title')}</h2>;
}
```

## Translation Files

Translation keys are defined in JSON files:

- **Source**: `src/assets/translations/en-US.json`
- **Types**: Auto-generated as `TranslationKey` type

### Adding New Keys

1. Add to `src/assets/translations/en-US.json`:

```json
{
    "componentName.title": "Component Title",
    "componentName.description": "This is a description",
    "componentName.button.submit": "Submit",
    "componentName.error.required": "This field is required",
    "componentName.items.count__one": "{{count}} item",
    "componentName.items.count__other": "{{count}} items"
}
```

2. Keys are automatically typed via `TranslationKey`

### Key Naming Convention

```
{domain}.{section}.{element}

Examples:
transactions.overview.title
transactions.filters.dateRange
transactions.table.column.amount
paymentLink.creation.form.amount.label
paymentLink.creation.form.amount.error.required
```

## Interpolation

### Simple Values

```json
{ "welcome.message": "Hello, {{name}}!" }
```

```typescript
i18n.get('welcome.message', { values: { name: 'John' } });
// → "Hello, John!"
```

### Multiple Values

```json
{ "transfer.summary": "Transfer {{amount}} to {{recipient}}" }
```

```typescript
i18n.get('transfer.summary', {
    values: {
        amount: i18n.amount(10000, 'EUR'),
        recipient: 'Account XYZ',
    },
});
// → "Transfer €100.00 to Account XYZ"
```

### Rich Text / HTML Markup — `<Translation />` Component

**Never** use `i18n.get()` to inject HTML or JSX into translation placeholders. Use the `<Translation />` component instead. It replaces `{{placeholder}}` with Preact nodes so you can wrap parts of a translated string in markup (bold text, links, icons, etc.) while keeping the full sentence translatable.

```typescript
import { Translation } from 'src/components/internal/Translation';
```

**Example — bolding part of a sentence:**

```json
{ "capital.overview.grants.item.details.requestedFunds": "You requested {{amount}} in funding" }
```

```tsx
<Translation
    translationKey="capital.overview.grants.item.details.requestedFunds"
    fills={{
        amount: (
            <Typography el={TypographyElement.SPAN} variant={TypographyVariant.BODY} strongest>
                {i18n.amount(grant.grantAmount.value, grant.grantAmount.currency)}
            </Typography>
        ),
    }}
/>
```

## Pluralization

Use `__one`, `__other` suffixes:

```json
{
    "items.selected__one": "{{count}} item selected",
    "items.selected__other": "{{count}} items selected"
}
```

```typescript
i18n.get('items.selected', { count: 1 }); // → "1 item selected"
i18n.get('items.selected', { count: 5 }); // → "5 items selected"
```

## Common Patterns

### Timezone-Aware Date Formatting

When displaying dates that should respect a specific timezone (e.g., a balance account's timezone), use the `useTimezoneAwareDateFormatting` hook instead of calling `i18n.date()` or `i18n.fullDate()` directly. The hook temporarily sets the timezone on the i18n instance for each formatting call and restores it afterward, preventing timezone leaks.

```typescript
import useTimezoneAwareDateFormatting from 'src/hooks/useTimezoneAwareDateFormatting';

const { dateFormat, fullDateFormat } = useTimezoneAwareDateFormatting(activeBalanceAccount?.timeZone);

dateFormat('2024-01-15T10:30:00Z');
// → "Jan 15, 2024" (in the balance account's timezone)
```

### Labels and Placeholders

```typescript
<InputText
    label={i18n.get('form.email.label')}
    placeholder={i18n.get('form.email.placeholder')}
/>
```

### Error Messages

```typescript
<FieldError message={i18n.get('form.email.error.invalid')} />
```

### Button Text

```typescript
<Button>{i18n.get('actions.submit')}</Button>
<Button>{i18n.get('actions.cancel')}</Button>
```

### Table Headers

```typescript
const columns = [
    { key: 'date', label: i18n.get('table.column.date') },
    { key: 'amount', label: i18n.get('table.column.amount') },
];
```

### Empty States

```typescript
{items.length === 0 && (
    <EmptyState message={i18n.get('componentName.empty.message')} />
)}
```

## Testing with Custom Translations

In stories, override translations:

```typescript
export const CustomLabels: ElementStory<typeof Component> = {
    args: {
        coreOptions: {
            translations: {
                en_US: {
                    'componentName.title': 'Custom Title',
                },
            },
        },
    },
};
```

## Checklist

- [ ] All user-visible text uses `i18n.get()`
- [ ] All amounts use `i18n.amount()`
- [ ] All dates use `i18n.date()` or `i18n.fullDate()`
- [ ] Keys follow naming convention
- [ ] Pluralization uses `__one` / `__other` suffixes
- [ ] New keys added to `en-US.json`
